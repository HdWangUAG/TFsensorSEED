# `decisions/` — sedimented discussion knowledge (the closed loop)

Auto-written by the **scribe** after a discussion (`minicrew sediment` or the
"📌 Sediment" button in History): consensus, decisions (owner + next step), open
questions, and candidate pitfalls — each note tagged with the `source_run` it
came from. Trust tier **MEDIUM** ("prior project decisions; revisit if new data
conflicts").

Crews that list `decisions` in their `knowledge:` automatically build on these,
so the system accumulates understanding across discussions instead of forgetting.
README.md is not injected. Candidate pitfalls here are for human review — promote
real ones into `pitfalls/` (HARD CONSTRAINT) yourself.
